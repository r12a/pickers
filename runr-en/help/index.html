This Unicode character app allows you to produce or analyse runs of Anglo-Saxon text using the Runic script. Character apps are especially useful for people who don't know a script well, as characters are displayed in ways that aid identification.

If something is broken or missing raise an issue. For release notes see the Github commit list. The pictures of the UI used in this page are taken from a variety of character apps.

Inputting characters

The app includes characters in the Unicode Runic block that are used for writing text in the Anglo-Saxon language.

To analyse text, copy & paste it into the large text area and select the desired control(s).

To produce Runic text in the text area, do one of the following:

  • Type on your keyboard, using type-assist. A quick way to input text from ASCII characters on your keyboard, based on a Latin transcription.
  • Click on the characters in the selection area. The blocks are arranged to assist with input, and groups of characters are arranged within a block phonetically.

The above are described in more detail elsewhere on this page.

Click on FORMAT to access and input invisible formatting characters used to control direction, joining forms, etc.

You can use the ⇐︎ • ⭅︎ below the text area to make the runic text run right-to-left, eg. ‮ᚹᚪᚱᚦᚷᚪ‬ (and if you have the right font, such as Junicode or a Babelstone font, make the glyphs face to the left also), or force the glyphs to be mirrored.

The Bablestone fonts also implement a number of bind-runes for Anglo-Saxon (but are missing some for Old Norse, it seems) if you put a ZWJ character between the characters you want to ligate. For example: ᚻ‍ᛖ‍ᛚ. You can also produce two glyphs mirrored around the central stave by putting ZWJ between two identical characters, eg. ᚢ‍ᚢ.  

Transliterate. Converts the contents of the text area to a Latin transliteration that provides a one-to-one correspondence between Runic and Latin symbols.

The transliteration scheme was developed for this character app, and, where possible, uses symbols that represent sounds typically associated with their Runic equivalents, rather than the sometimes fairly arbitrary selection of diacritics found in other systems. Of course, the complexities of mapping graphemes to sounds means that this is far from a real phonetic transcription, and it doesn't show vowels for unvowelled text, but the process is reversible (unlike most transcriptions).

Transliterations produced this way are particularly useful for understanding the composition of text in scripts, where the individual parts are not well separated in the text.

Make vocab. This helps speed up the creation of entries for the vocabulary app. You can create a vocab item with up to 5 parts, delimited by / or |, in the following order:

  1. native text
  2. meaning
  3. transcription (IPA or other, optional)
  4. alternative transcription (if needed)
  5. notes (optional)

The basic transcription should be in IPA, wherever possible. You can also add an alternative transcription. This will be styled differently from the IPA.

Much of the time, only the first 2 items are needed. In this app if an alternative transcription is not supplied it will be added automatically, though it should be checked carefully.

Here is an example of before and after.

ᚫᛣᚱᛁᛋᛏ/Christ

will produce:

ᚫᛣᚱᛁᛋᛏ|Christ|(ækrist)

The app will also, simultaneously, produce markup for examples in the secondary text area (see below).

If you have more than one line of text in the text area, you must highlight the line you want to convert. Only convert one line at a time.

Vocab to markup. Converts a vocab entry to markup that can be used for an example. Must be applied to one line at a time.

មេឃ|sky|meːk

will produce:

<span class="charExample" translate="no"><bdi class="ex" lang="km" dir="ltr">មេឃ</bdi> <bdi class="trans">m̱eḵʰ</bdi> <bdi class="ipa">meːk</bdi> <bdi class="meaning">sky</bdi></span>

Latin to Runic. Converts a Latin transliteration into Runic text, using the Anglo-Saxon/Frisian Fuþorc. Symbols used in the various sources can vary, so the result needs to be checked for appropriateness.

Translit+. Converts Anglo-Saxon/Frisian Fuþorc text to a Latin transliteration using the transliteration algorithm, then changes eᵃ to e͡a. This function provides only a general indication of the transliterated Runic.
